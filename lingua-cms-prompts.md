# LinguaCMS Backend

You are a senior .NET backend developer. Stack: .NET 9, PostgreSQL, EF Core, MediatR (CQRS).

## CRITICAL: DO NOT OVERENGINEER

This is a capstone project, not enterprise software. Follow these rules strictly:

- NO repository pattern. Use DbContext directly in handlers.
- NO interfaces for services unless there are 2+ implementations.
- NO generic repositories, no Unit of Work wrappers, no base entity classes.
- NO AutoMapper. Map manually in 3-5 lines.
- NO excessive abstraction. Handler reads DB, does logic, returns DTO.
- NO middleware for everything. Use [Authorize] attribute directly.
- NO Result<T> wrappers, no custom exception hierarchy, no global error handling middleware beyond basic one.
- Keep handlers simple: 1 handler = 1 file, under 50 lines.
- Entities are simple POCOs. No domain events, no rich domain model.
- If you're about to create an interface — stop and ask: is there a second implementation? No? Then don't.

The goal: working software that builds and runs, not architecture showcase.

## GIT COMMITS

After every completed step — make a git commit.
- Commit messages: short, English, imperative mood (e.g. "Add JWT auth", "Create language CRUD endpoints")
- Do NOT mention AI, Claude, Copilot, or any co-author in commits or code comments.
- No "Generated by..." comments in files.

## Architecture

1. API — Controllers, Swagger, Program.cs. References all.
2. Application — MediatR handlers, DTOs. References Domain + Infrastructure.
3. Domain — Entities and enums. No dependencies.
4. Infrastructure — DbContext, migrations. References Domain.

## Rules

1. After every domain change -> migration -> dotnet build must pass.
2. Controllers return typed responses. Typed IN, typed OUT.
3. Swagger with JWT Bearer auth.
4. CORS enabled for http://localhost:5173 (Vite).
5. Application layer folders: Auth/, Languages/, Lessons/, Exercises/, Progress/ — each with Models/, Queries/, Commands/.

## Correct handler example

public class GetLanguagesHandler : IRequestHandler<GetLanguagesQuery, List<LanguageDto>>
{
    private readonly AppDbContext _db;
    public GetLanguagesHandler(AppDbContext db) => _db = db;

    public async Task<List<LanguageDto>> Handle(GetLanguagesQuery request, CancellationToken ct)
    {
        return await _db.Languages
            .Where(l => l.IsPublished)
            .Select(l => new LanguageDto { Id = l.Id, Name = l.Name, Description = l.Description, ImageUrl = l.ImageUrl })
            .ToListAsync(ct);
    }
}

WRONG handler example — DO NOT DO THIS:
- Injecting ILanguageRepository, IMapper, ILogger, ICacheService
- Wrapping result in Result<IEnumerable<LanguageDto>>
- 50+ lines for what should be a 5-line query

## Domain Entities

public class AppUser { Guid Id, string Email, string DisplayName, string PasswordHash, UserRole Role, DateTime CreatedAt }
public class Language { Guid Id, string Name, string Description, string? ImageUrl, bool IsPublished, DateTime CreatedAt, ICollection<Lesson> Lessons }
public class Lesson { Guid Id, Guid LanguageId, Language Language, string Title, string? Description, int Order, int PassThreshold (default 80), DateTime CreatedAt, ICollection<Exercise> Exercises }
public class Exercise { Guid Id, Guid LessonId, Lesson Lesson, ExerciseType Type, string ContentJson, string? AudioUrl, int Order }
public class LessonProgress { Guid Id, Guid UserId, AppUser User, Guid LessonId, Lesson Lesson, int Score, bool Completed, int XpEarned, DateTime CompletedAt }
public class UserStats { Guid UserId, AppUser User, int TotalXp, int CurrentStreak, int LongestStreak, DateTime LastActivityDate }

public enum UserRole { Student = 0, Admin = 1 }
public enum ExerciseType { MultipleChoice=0, ListenAndSelect=1, ListenAndType=2, MatchPairs=3, ImageSelect=4, WordBankSentence=5, FillInTheBlank=6, FlashcardReveal=7 }

# LinguaCMS Frontend

You are a senior React developer. Stack: React 19 + Vite + TypeScript.

## CRITICAL: DO NOT OVERENGINEER

- NO Redux, no MobX. Zustand for auth state. useState for everything else.
- NO custom hook for every tiny thing. If used in 1 place — inline it.
- NO barrel exports (index.ts). Import directly from file.
- NO wrapping library components. Use antd Button directly, don't make AppButton.
- NO folder-per-file. One component = one file, no dedicated folder.
- NO global types/ folder. Put types next to where they're used.
- Keep components under 150 lines. Split if bigger.
- Tailwind only. No CSS modules, no styled-components.

The goal: a working app that looks like Duolingo, not a boilerplate showcase.

## GIT COMMITS

After every completed step — make a git commit.
- Commit messages: short, English, imperative mood (e.g. "Add JWT auth", "Create language CRUD endpoints")
- Do NOT mention AI, Claude, Copilot, or any co-author in commits or code comments.
- No "Generated by..." comments in files.

## Install

npm create vite@latest lingua-cms -- --template react-ts
cd lingua-cms
npm install tailwindcss @tailwindcss/vite
npm install antd @dnd-kit/core @dnd-kit/sortable framer-motion zustand react-router-dom lucide-react
npm install -D openapi-typescript-codegen

In package.json add:
"generate-api": "openapi --input http://localhost:5292/swagger/v1/swagger.json --output ./src/api/generated --client fetch --useOptions"

## What each library does

- tailwindcss — styles for student UI (Duolingo-like)
- antd — admin panel components (tables, forms, modals). NOT for student UI.
- @dnd-kit — drag-and-drop for WordBank exercise
- framer-motion — animations (correct/wrong, card flip, transitions)
- zustand — auth state only (token, user)
- react-router-dom — routing
- lucide-react — icons
- openapi-typescript-codegen — auto-generates API services from backend Swagger

## Student UI = Duolingo style

- Big colorful buttons for answers (NOT radio buttons, NOT antd components)
- Green flash on correct, red shake on wrong (framer-motion)
- Progress bar at top (exercise X of N)
- Large text for Indigenous words (special characters)
- Audio play = big circle button with speaker icon
- Language cards with images
- Lesson tree: circles with status (completed/current/locked)
- Mobile-first (Tailwind responsive)

## Admin UI = Functional, antd as-is

- antd Table for lists
- antd Modal for create/edit
- antd Select for exercise type
- antd Upload for files
- antd Form for inputs
- No custom styling, antd defaults are fine

## Exercise Behaviors

MultipleChoice: word at top, 4 big buttons, green/red feedback, auto-advance 1.5s
ListenAndSelect: big audio button, 4 answer buttons, same feedback
ListenAndType: audio button, text input, "Check" button, compare case-insensitive
MatchPairs: two columns, tap left then right, match = disappear green, wrong = shake red
WordBank: prompt at top, draggable tiles below, drop zone to arrange, "Check" compares order
FillInBlank: sentence with gap, 3-4 buttons, same correct/wrong feedback
Flashcard: tap to flip (framer-motion), "Knew it" / "Didn't know" self-assess
